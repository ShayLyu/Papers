## 摘要

灵巧操作仍然是机器人学中最具挑战性的问题之一，因为它要求在复杂、接触丰富的动力学条件下，对高自由度机械手和机械臂进行协调控制。一个主要障碍是本体差异：不同灵巧手具有不同的运动学和动力学特性，使以往方法不得不为每种本体分别训练策略，或依赖共享动作空间并为每种本体配置独立解码头。本文提出 DexFormer，一种端到端、具备动力学感知能力的跨本体策略。它以改造后的 Transformer 为骨干网络，并以历史观测作为条件。DexFormer 利用时间上下文在线推断形态与动力学，从而适应多样化的手部配置，并生成与本体相匹配的控制动作。在多种程序化生成的灵巧手资产上训练后，DexFormer 学得了可泛化的操作先验，并能强力零样本迁移到 LEAP Hand、Allegro Hand 和 RAPID Hand。实验表明，单一策略能够跨异构手部本体泛化，为跨本体灵巧操作建立了可扩展的基础。项目网站：https://davidlxu.github.io/DexFormer-web/。

## I. 引言

灵巧操作是通用机器人操作的一项关键能力。多指机器人手可以实现广泛的功能性行为，包括举起并重新定向日常物体 [13,18,29]、在杂乱场景中抓取 [25,5]、非抓持操作 [3,22]，以及手内重定向和精细姿态调整 [12,4,21,27]。然而在实践中，每迁移到一种新的手部本体，通常都需要从头精心训练和调试策略。不同的手具有不同的运动学、惯量、执行器响应与接触动力学，因此同一种控制策略可能产生截然不同的行为。这种本体特定的动力学使学习和仿真到现实迁移变得脆弱且昂贵；每换一个新平台，往往都需要重复进行系统辨识、重新调参，或额外采集真实世界数据 [12,4]。

即便对于单个机器人，仿真到现实的鸿沟也已经很大；当迁移发生在不同灵巧手本体之间时，这一问题会更加严重。现有弥合该鸿沟的方法大致分为三类。第一类是系统辨识（SysID），即通过数据驱动的模型辨识或主动探索，使仿真器参数拟合真实硬件，从而减少仿真与现实之间的参数失配 [9,19]。SysID 虽然有效，却受限于所选模型参数化形式，并且通常需要针对每个新本体重复执行。第二类是域随机化：在广泛的物理与形态参数分布上训练策略，使真实世界只是训练分布中的另一个样本。大规模随机化已经在无需采集真实数据的情况下实现了灵巧手内操作的仿真到现实迁移 [2]；在程序化生成机器人上进行的大规模强化学习，也已经产生了通用运动策略，可以在没有显式运动学知识的情况下零样本控制此前未见过的机器人 [11]。但是，单纯随机化并没有提供一种在测试时显式补偿本体特定动力学的机制。

第三类方法在仿真策略之上学习残差模型或补偿模型，以处理未建模动力学。例如，文献 [6] 学习无监督执行器网络，用于捕捉硬件的非线性效应。类似地，DexNDM [12] 从真实交互数据中学习逐关节神经动力学模型，并在仿真策略之上训练残差控制器，从而在无需精确估计物体状态的情况下实现稳健的灵巧操作。这些残差方法可以显著缩小仿真到现实的差距，但依赖于采集特定本体的真实数据，因此并未直接解决零样本跨本体迁移问题。

本文提出 DexFormer：一种基于历史条件 Transformer 的跨本体灵巧操作策略。DexFormer 利用时间上下文，从观测历史中进行隐式形态推断，使策略无需显式形态标识符或本体专用网络头，就能在线适应不同手部动力学。我们通过对标准灵巧手进行程序化随机化，构建了广泛的形态分布，并证明在随机化本体上训练的单一、形态无关策略，可以零样本泛化到未见过的标准手及其变体。因此，DexFormer 为跨本体灵巧性提供了一条可扩展路径，同时揭示了大规模强化学习训练与能够学习潜在物理结构的历史条件模型之间很有前景的联系。

**图 1.** DexFormer 学习统一的历史条件策略，将灵巧抓取迁移到多种手部本体，实现从大规模仿真到真实机器人的零样本部署。

本文主要贡献如下：

- 提出 DexFormer：一种在大规模手部本体上训练的历史条件 Transformer，可执行隐式形态推断，实现跨本体自适应控制；
- 开发大规模形态随机化流水线与分布式训练框架，用于灵巧手跨本体训练；
- 证明单个形态无关策略无需手工重定向、显式形态编码或独立策略头，即可泛化到未见过的灵巧本体和多种真实灵巧手。

## II. 相关工作

### A. 跨本体灵巧操作

跨本体灵巧操作主要沿两个方向展开。一类工作先生成跨手型的静态抓取位姿，再进行开环执行；它们利用统一的接触表示、优化方法或物理求解器，在不同形态上生成可行抓取 [23,24,28]。然而，这类开环策略难以在执行过程中响应扰动与实时接触变化。另一类工作学习本体感知的闭环控制器：或显式编码手部运动学图，并将专家策略蒸馏为单个零样本策略 [15]；或定义特征抓取动作空间，通过重定向映射将策略迁移到新手型 [26]。这些方法依赖显式运动学建模或重定向映射，难以扩展到结构根本不同的标准手型。相比之下，我们在标准共享动作空间中学习历史条件策略，从时间上下文中隐式推断本体特定动力学，从而无需显式形态编码或重定向，即可在异构手型之间实现闭环零样本迁移。

### B. 动力学感知操作

这一方向的方法显式建模或补偿仿真与真实动力学之间的失配。无监督执行器网络（UAN）[6] 从真实轨迹中学习对仿真器执行器模型的数据驱动修正，通过减少高动态移动操作中对仿真器误差的利用来改善仿真到现实迁移。DexNDM [12] 则从自主采集的真实交互数据中学习逐关节神经动力学模型，并用它在仿真策略之上训练残差控制器，从而在无需精确估计物体状态的情况下，对多种物体实现稳健的手内旋转。作为补充，Zhao 等人 [30] 通过经校准的执行器和触觉动力学丰富仿真，将高保真触觉仿真、电流-力矩标定和随机化电机非理想因素结合起来，训练可零样本部署到真实硬件的力感知策略。这些方法依赖显式的真实动力学学习或高精度仿真器建模。我们的方法无需额外动力学模型或残差适配，而是在广泛随机化的形态与动力学分布上训练单一策略，让时间上下文隐式捕捉本体特定行为。

### C. 测试时适应

测试时适应和在线适应已经在足式运动中得到广泛研究，并开始用于操作任务。快速运动适应（RMA）[7] 从近期状态-动作历史中在线学习潜在嵌入，并以该嵌入作为基础策略的条件，从而无需额外真实世界试验或标定，就能快速补偿未建模动力学。这一思路的变体已应用于双足机器人 [8]、机械臂 [10] 和灵巧手内旋转 [16]，表明短时间历史能够支持快速系统辨识，并在接触与负载变化时进行闭环修正。与此同时，LocoFormer [11] 等序列模型中的上下文内适应方法，直接以历史作为策略条件，从而无需显式辨识模块，即可跨地形和任务实现零样本适应。在我们的设定中，并不为每个环境显式学习独立适应模块；相反，我们在程序化随机化的广泛手部形态分布上学习一个历史条件策略，使其从时间上下文中隐式推断本体特定动力学，并零样本泛化到未见过的手部本体。

## III. 方法

### A. 问题定义

我们将灵巧操作建模为一个部分可观测马尔可夫决策过程（POMDP）：机器人手必须抓取并稳定操作从任务分布中采样的物体。每个回合采样一个物体 `o ∈ O` 和一个手部本体 `e ∈ E`，由此产生依赖形态的状态转移动力学。

目标是学习形态无关策略 `πθ`。该策略接收观测，但无法获知本体身份。令 `o_t` 表示时刻 `t` 的系统观测，包括手、机械臂与物体点云；令 `a_t` 表示策略在时刻 `t` 选择的动作。强化学习目标是在本体与物体的联合分布上最大化折扣回报：

```text
max_θ E_{e~p(E), o~p(O), τ~πθ} [ Σ_{t=0}^T γ^t R(o_t, a_t) ].    (1)
```

其中，`R` 奖励成功抓取行为。由于本体不可观测，智能体必须通过时间交互推断依赖形态的动力学，并相应调整控制。

### B. 共享动作空间

考虑一组具有不同驱动手指关节数量的灵巧机器人手。令 `E` 为手部本体集合；对每个本体 `e ∈ E`，令 `d_e` 为其驱动手指关节的自由度数。为在异构手型之间建立统一控制接口，我们定义一个 `D_F` 维标准手指动作空间，其中具有相同功能角色的关节（例如拇指外展、食指屈曲）在不同本体之间被分配到相同的标准索引。

对于具有 `d_e` 个驱动手指关节的本体 `e`，其原生手指指令记为 `ã_t^F(e) ∈ R^{d_e}`。通过固定嵌入算子

```text
P_e: R^{d_e} -> R^{D_F}
```

将其嵌入标准动作空间。该算子按照 MCP 外展与屈曲等功能角色，将 `d_e` 个关节指令写入本体特定的标准索引，并将其余项置零：

```text
a_t^F = P_e(ã_t^F(e)) ∈ R^{D_F}.
```

自由度较少的本体对未使用的标准维度进行零填充，而高自由度手可以占用 `D_F` 中全部或大部分索引，如图 2 所示。我们进一步使用时间平滑得到实际执行的手指动作。令 `â_t^F ∈ R^{D_F}` 为策略输出的原始手指动作，则平滑后的手指指令定义为：

```text
a_t^F = λ â_t^F + (1 - λ) a_{t-1}^F,    (2)
```

其中 `λ ∈ (0,1]` 控制动作平滑程度。时刻 `t` 的完整高层动作是：

```text
a_t := [a_t^F, a_t^A] ∈ R^{D_F + D_A},    (3)
```

其中 `a_t^A ∈ R^{D_A}` 是机械臂末端位姿增量（例如七自由度机械臂取 `D_A = 7`）。这一共享高层动作由本体专用的底层控制器接收，并映射为力矩或关节目标；对于自由度较低的手，零填充的手指维度会被直接忽略。第 IV 节实验给出了 `D_F` 的具体取值和所用手部本体。

**图 2. 共享动作空间。**  
(a) 标准动作空间定义形态不变嵌入：具有相同解剖功能的关节共享固定动作索引。MCP（蓝/橙）负责屈曲与外展；PIP/DIP（绿）提供近端/远端屈曲；拇指采用不同结构，其中 CMC（橙/紫）支持外展和对掌，MCP（蓝）控制基部屈曲，IP（绿）负责远端屈曲。  
(b) 标准嵌入展平为 20 维空间。LEAP 和 Allegro 等低自由度手对未使用维度进行零填充，RAPID Hand 等高自由度本体则填满整个嵌入，从而实现异构本体间的共享控制。

### C. 本体生成

我们通过扰动标准手的形态相关物理参数，构建灵巧手本体族。令 `E_canon` 表示标准本体集合；对每个 `e ∈ E_canon`，令 `z(e)` 表示其形态参数，包括连杆长度、质量与惯量。随机化本体按下式采样：

```text
z^(k)(e) ~ p_morph(z | e),  k = 1, ..., N_e.
```

其中 `p_morph` 是定义在形态参数上的扰动分布。每个采样形态 `z^(k)(e)` 产生一个新本体 `e^(k)`，同时保留原始运动学图拓扑与驱动结构。这些本体的并集

```text
E_train = ⋃_{e∈E_canon} {e^(1), ..., e^(N_e)}
```

构成强化学习所用的形态丰富训练集。在 `E_train` 上训练使策略接触到连续的物理实现空间，从而提升稳健性，并改善评估时对未见标准本体的零样本迁移。

### D. 历史条件 Transformer

为了推断无法从单个状态观测到的潜在形态参数，我们令策略以有限历史窗口为条件。令 `h_t` 表示时刻 `t` 可用的历史，由过去的观测-动作对和当前观测组成。原文记为：

```text
h_t = {o_k}_{k=t-H+1}^t.
```

我们将这段历史处理为 `H` 个时间 token 的序列。对每个 `k ∈ [t-H+1, t]`，使用观测 `o_k` 的嵌入构造 token。加入可学习的位置编码后，将序列输入 Transformer 编码器。关键在于，我们对自注意力应用因果掩码，保证任意 token `k` 只能关注先前 token `j ≤ k`，防止未来信息泄漏。Transformer 输出一系列上下文化潜在嵌入。对应当前时刻 `t` 的最后一个嵌入作为历史的紧凑摘要，随后传入 MLP 动作头，产生 `a_t` 的动作分布，如图 3 所示。

**图 3.** 历史条件 Transformer 策略架构。每个时刻输入长度为 `H` 的固定观测历史，并将其 token 化为 `H` 个 token。该序列经过带位置编码和因果自注意力的三层 Transformer。取能够关注全部先前历史的最终嵌入表示，传入 MLP 动作头以参数化随机策略，再从所得分布中采样动作并执行。

### E. 观测空间

策略观测空间定义为：

```text
o_t := [o_t^OC, o_t^OT, a_{t-1}, o_t^JP, o_t^JV, o_t^FT, o_t^FC, o_t^V] ∈ R^377.    (4)
```

其中，当前物体四元数 `o_t^OC ∈ R^4`，目标物体位姿 `o_t^OT ∈ R^7`，上一时刻动作 `a_{t-1} ∈ R^27`，关节位置 `o_t^JP ∈ R^27`，关节速度 `o_t^JV ∈ R^27`，指尖与手掌状态 `o_t^FT ∈ R^78`，接触力 `o_t^FC ∈ R^15`，物体点云 `o_t^V ∈ R^192`。

### F. 奖励设计

总奖励由五项组成：

```text
r_t = r_t^P + r_t^D + r_t^M + r_t^T + r_t^S.    (5)
```

其中，`r_t^P` 惩罚过大的动作以确保控制平滑；`r_t^D` 鼓励末端执行器接近物体；`r_t^M` 鼓励手指抓住物体；`r_t^T` 在有接触时引导位姿跟踪；`r_t^S` 在无接触时引导位姿跟踪。

**表 I. 奖励项及其公式。**

| 项 | 公式/含义 |
| --- | --- |
| `r_P` | `-λ1 Σ_i a_i^2 - λ2 Σ_i (a_i - a_i^prev)^2` |
| `r_D` | `λ3 (1 - tanh(max ||p_ee - p_obj||_2 / σ))` |
| `I_m` | 指示至少一个手指接触力超过阈值：`||f_thumb||>τ` 或 `||f_index||>τ` 或 `||f_middle||>τ` 或 `||f_ring||>τ` |
| `I_c` | 指示有效抓握接触：拇指接触且食指/中指/无名指至少一个接触 |
| `r_M` | `λ4 I_m + λ5 I_c` |
| `r_T` | `λ6 (1 - tanh(||p_obj - p_des||_2 / σ_p0)) I_c` |
| `r_S` | `λ7 (1 - tanh(||p_des - p_obj||_2 / σ_p1))^2` |

这里，`a_i` 和 `a_i^prev` 表示当前与上一时刻动作，`p_ee` 和 `p_obj` 表示末端执行器与物体位置，`I_c` 表示有效接触，`I_m` 鼓励接触物体；`f_thumb`、`f_index`、`f_middle` 和 `f_ring` 分别是拇指、食指、中指和无名指指尖测得的接触力向量；`τ = 1` 为力阈值；`d_q(·)` 衡量四元数距离。超参数为：`λ1=0.005`、`λ2=0.005`、`λ3=2`、`λ4=0.8`、`λ5=2`、`λ6=14.0`、`λ7=20`、`σ_p0=0.2`、`σ_p1=0.1`。

### G. 跨本体训练的并行化

Isaac Lab 等仿真平台支持在单个 GPU 上并行运行环境，但通常要求同一 GPU 内的运动学拓扑和执行器结构相似，而不同手部本体并不满足这一点。为适应这一限制，我们将同一标准本体的随机化变体分组到单个 GPU 上，并将形态上不同的标准本体分配到不同 GPU。每个 GPU 独立执行 rollout；同时使用 Distributed Data Parallel（DDP）维护单一共享策略：在轨迹采集和前向传播期间，每个 GPU 在本地运行；在反向传播期间，通过 all-reduce 同步梯度，以确保参数更新一致。该配置在 GPU 间并行化形态多样性，并在 GPU 内并行化环境，从而支持单一 DexFormer 策略的可扩展跨本体训练，如图 4 所示。

**图 4.** 分布式训练中的梯度聚合。rollout 和前向传播在独立 GPU 上本地计算；反向传播时，all-reduce 原语聚合梯度并平均参数更新，确保所有设备上的 DexFormer 权重保持一致。

## IV. 实验

本节展示 DexFormer 在未见本体上的仿真与真实世界评估。实验旨在系统回答以下研究问题：

1. DexFormer 的抓取成功率与 GRU/LSTM 基线相比如何？
2. DexFormer 在异构手之间的零样本泛化能力如何？
3. 历史上下文是否提升策略性能？
4. 由训练集中本体数量衡量的本体多样性如何影响零样本性能？
5. 性能是否随训练并行度（环境数量）扩展？
6. DexFormer 对锁定单个关节造成的形态损伤有多稳健？

### A. 实现细节

- Franka-Leap-Canonical

训练总共使用 300 个随机化本体，即每个标准手 100 个变体，如图 5(a) 所示。这些变体构成形态丰富的训练分布，用于学习单一跨本体策略。评估在两个设定下进行：(i) 零样本迁移到三个标准本体；(ii) 泛化到从同一形态分布采样的 32 个未见本体，如图 5(b) 所示。对每个并行仿真环境，随机从预定义的 10 个物体中采样一个物体，如图 6 所示。

### B. 指标与基线

策略性能使用成功率评估，即目标物体到达指令目标位置的试验比例。成功条件为：

```text
||p_goal - p_target||_2 < 0.05.
```

DexFormer 与两个使用显式隐藏状态编码时间上下文的循环策略基线比较：

- **LSTM 基线：** 使用隐藏大小为 128 的三层 LSTM 作为时间记忆模块，跨时间步维护 cell state 和 hidden state，以捕捉观测历史中的长程依赖。
- **GRU 基线：** 使用隐藏大小为 128 的三层 GRU，以更轻量的门控循环结构替代 LSTM，并用单一隐藏状态建模时间依赖。

## V. 结论

本文提出 DexFormer，一种用于跨本体灵巧操作的历史条件 Transformer 策略。通过以观测-动作历史为条件，DexFormer 隐式推断形态与动力学，使单一策略无需本体标识符或专用解码头，即可跨异构机器人手泛化。大量实验表明，DexFormer：(i) 优于基于 GRU 和 LSTM 的基线；(ii) 在 LEAP、Allegro 和 RAPID 手及其变体之间实现强零样本迁移；(iii) 受益于时间上下文、形态多样性和环境级并行度。这些发现说明，动力学感知的时间推断是跨本体泛化的一种有效机制。真实世界评估进一步支持该方法的实用性，表明形态无关的灵巧操作可以扩展到仿真之外。总体而言，DexFormer 为适用于多样化本体与部署场景的统一操作策略提供了一条可扩展路径。

## VI. 局限性

虽然 DexFormer 在本体变化上表现出强跨本体泛化能力，但仍存在若干局限。首先，当前训练制度聚焦于有限的物体几何和质量属性，导致物体层面的泛化不足。将训练扩展到更大、更多样的物体集合，并引入多专家蒸馏或混合专家模型，可能改善其对真实世界操作场景的稳健性与可迁移性。

其次，在 RAPID Hand 等更高自由度本体的零样本设置下，作者观察到轻微性能下降。这表明更复杂的架构或扩展策略可能有益，包括更深的时间建模、改进的动作嵌入，以及更高程度的环境级并行。此外，由于 Isaac Lab 当前限制，具有不同运动学拓扑的本体被训练在不同 GPU 上。支持跨本体混合缓冲区或集中式梯度累积，可能降低梯度方差并提升泛化。

作者认为，解决这些局限是未来工作的重要方向，尤其是面向灵巧操作的 foundation-style 模型。更广泛地说，作者希望推动一种跨本体学习范式：可扩展的强化学习训练使高自由度、形态多样的本体能够大规模训练，而下游用户可以通过后训练、微调或引入残差策略进一步适应新本体与新任务。作者将这一路径视为迈向更通用跨本体操作策略的一步。
