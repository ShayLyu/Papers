## 摘要

由于高维状态空间和动作空间的复杂性，灵巧抓取仍然是机器人领域的核心挑战之一。本文提出 T(R,O) Grasp，这是一个基于扩散的框架，能够在多种机器人手之间高效生成准确且多样的抓取。其核心是 T(R,O) Graph：一种统一表示，用于建模机器人手与物体之间的空间变换，同时编码二者的几何属性。结合高效的逆运动学求解器，图扩散模型同时支持无条件抓取合成和有条件抓取合成。

在多种灵巧手上的大量实验表明，T(R,O) Grasp 在 NVIDIA A100 40GB GPU 上达到 94.83% 的平均成功率、0.21 秒的推理速度，以及每秒 41 个抓取的吞吐量，显著优于现有基线。此外，该方法在不同本体之间具有鲁棒性和泛化能力，并显著降低内存消耗。更重要的是，高推理速度使闭环灵巧操作成为可能，表明 T(R,O) Grasp 具备扩展为灵巧抓取基础模型的潜力。代码、附录和视频见：https://nus-lins-lab.github.io/trograspweb/ 。

## I. 引言

使用灵巧手进行抓取，是实现精确、类人操作的一项基础能力。然而，高效生成多样且高质量的抓取仍然是长期存在的挑战，主要原因在于灵巧手的高维度，以及同时保证稳定性和精确性的困难。现有学习式研究主要探索了两类表示：以机器人为中心的表示和以物体为中心的表示。

以机器人为中心的方法 [2]-[6] 通常用腕部位姿或关节值表示抓取，并通过将观测直接映射为控制信号实现快速推理。然而，由于学习到的表示隐式绑定于特定手部本体，它们往往样本效率较低，并且迁移能力有限，难以泛化到新的形态。相反，以物体为中心的方法用接触点或可供性等物体特征描述抓取 [7]-[14]，天然更有利于跨物体和跨机器人平台迁移。但这类表示在部分观测下常较脆弱，并且通常需要额外优化阶段，例如指尖逆运动学或接触图的约束拟合，才能把抽象预测转换为可行关节配置，因此计算代价高且耗时。

近期，D(R,O) Grasp [1] 为灵巧抓取引入了一种以交互为中心的表示。不同于以机器人或物体为中心的方法，交互中心表示通过直接建模机器人手与物体之间的空间关系来刻画抓取，并把二者嵌入共享的交互空间。这种形式结合了机器人中心与物体中心范式的优点：既保留本体感知能力，也捕获可迁移的物体层级特征。因此，它有助于跨不同手部本体和物体形状泛化，保留潜在接触的细粒度几何结构，并在部分观测下更加鲁棒。

在 D(R,O) 中，这一目标通过构建手部采样点与物体采样点之间的距离矩阵来实现。然而，D(R,O) Grasp 中的点到点特征图需要大量 GPU 内存，导致表示资源开销大、难以扩展。此外，D(R,O) 依赖合适的初始状态作为输入，使模型性能容易受到不可行初始化的影响。

为克服 D(R,O) 表示的限制，本文提出 T(R,O) Grasp：一种基于 T(R,O) Graph 表示的高效图扩散模型，用于跨本体灵巧抓取合成。形式上，对于任意物体-手部组合，我们构建机器人与物体之间的统一变换图，即 T(R,O) Graph。在该图中，节点编码物体与连杆几何，以及抓取时它们在物体全局坐标系中的空间变换；边则由节点之间的相对空间变换组成。

相比机器人中心表示，这一定义在所有物体和灵巧手本体之间是统一且可泛化的。不同于 D(R,O) 所需的内存密集型点到点表示，T(R,O) Graph 在物体 patch 与手部 link 之间建立关系，显著降低内存使用，并加快训练和推理。基于 T(R,O) 表示，我们实现了一个高效的 Transformer 图扩散模型来生成 T(R,O) Graph，从而支持无条件或基于手部状态的有条件跨本体灵巧抓取合成。

在大量实验中，T(R,O) Grasp 达到 94.83% 的抓取成功率，在 NVIDIA A100 40GB GPU 上平均推理速度为 0.21 秒，吞吐量为每秒 41 个抓取。

本文主要贡献如下：

1. 提出 T(R,O) Graph：一种统一且可泛化的表示，用于建模所有物体-手部组合之间的交互。与 D(R,O) 表示相比，它显著提高内存效率，并为扩展到大规模灵巧抓取基础模型奠定基础。
2. 在 T(R,O) Graph 上构建高效图扩散模型，同时支持有条件和无条件抓取合成。该设计不仅缓解了对不可行初始化的敏感性，也使可靠的闭环灵巧操作成为可能。
3. 在仿真和真实世界中开展广泛实验，证明 T(R,O) Grasp 在灵巧抓取成功率上显著优于所有基线，并比现有方法具有更高效率。

## II. 相关工作

### A. 灵巧抓取表示

使用灵巧机器人手进行抓取，在实现类人精度操作方面具有巨大潜力。然而，由于其内在高维性，这仍然极具挑战。要实现跨本体灵巧抓取的统一模型，需要一种高效且鲁棒的表示，能够一致编码物体与机器人手的几何及交互。

一种自然直接的方法是用手部关节值表示抓取 [2]-[4], [15]。虽然这些预测可以直接执行，但关节值表示缺乏规范定义，因为同一只手可能存在多个有效关节参数化。此外，它跨本体的泛化能力有限，高维动作空间中的强化学习计算成本也限制了效率。

另一类研究以物体为中心表示抓取，通常编码物体接触点或接触图 [7]-[12], [16]。尽管这些方法可以跨本体和物体泛化，但通常依赖基于接触的优化，阻碍实时抓取合成。近期，D(R,O) Grasp [1] 率先引入基于交互的表示，通过预测物体与机器人手之间的点到点距离矩阵，显著促进了跨多样物体和手部本体的鲁棒泛化。然而，D(R,O) 表示仍存在限制：其一，D(R,O) 估计前构建的点到点特征图带来大量内存开销，内存效率低且不利于扩展；其二，预测高度依赖初始位姿，因为它条件化于 CVAE [17] 的潜在输出，当初始位姿较差或不可行时，性能会下降。

本文提出一种新的轻量且高效的表示，围绕手部 link 与物体 patch 之间的空间变换展开。同时，我们实现了轻量图扩散模型，支持无条件和有条件抓取合成。

### B. 使用生成模型进行抓取合成

随着生成模型的成功，大量研究在大规模数据集上训练生成器，并结合样本优化进行抓取合成。Contact-GraspNet [18] 可在杂乱场景中高效生成无碰撞抓取；GraspME [19] 引入流形估计器，在物体流形上建模可行抓取；Neural Grasp Distance Fields [20] 利用隐式距离表示进行连续抓取合成；SE(3)-DiffusionFields [21] 进一步使用扩散学习，通过平滑代价函数联合优化抓取与运动。

尽管取得进展，既有方法仍局限于单个机器人夹爪的 6 自由度位姿，而这对灵巧手抓取而言从根本上不足，因为缺少合适表示。本文提出的 T(R,O) Graph 为具有任意几何和 link 数量的灵巧手提供统一且轻量的表示，使生成模型能够进行远超 6 自由度夹爪的高维抓取合成。

## III. 方法

给定物体点云 \(P^O \in R^{N \times 3}\)、灵巧手的 URDF 描述，以及每个 link 局部坐标系中的 L 个 link 点云 \(\{P_i^R\}_{i=1}^{L}\)，T(R,O) Grasp 首先预测所有 link 的 SE(3) 变换，并暂时放松关节约束，从而生成多样抓取。随后，预测变换通过基于 Pyroki 的逆运动学 [22] 被优化并转换为可执行关节值，在此过程中施加关节约束以保证可行性。

### 方法概述

图 2 展示了本文框架。我们将灵巧抓取合成形式化为所提出 T(R,O) Graph 的去噪过程。由于空间变换在不同物体和手部本体之间具有一致定义，该图可作为基于扩散的生成过程的统一表示，其中带噪 link 位姿被逐步去噪为有效抓取。此外，本文训练与推理流程支持无条件抓取合成，以及由初始手部状态引导的有条件抓取合成。

### A. T(R,O) Graph

学习跨本体灵巧抓取需要一种统一表示，用来刻画灵巧手与物体之间的关系。与不同本体之间差异很大的关节值相比，空间变换为跨不同物体和本体提供了更鲁棒、更一致的表示。因此，本文围绕物体 patch 与灵巧手 link 的空间变换构建 T(R,O) Graph。

在 T(R,O) Graph 中，节点表示每个物体 patch 或手部 link 的几何属性，以及抓取时它相对于物体全局坐标系的空间变换。边编码物体 patch 与手部 link 之间、以及手部 link 之间的相对变换。

由于 SE(3) 是非线性流形，无法直接应用标准扩散过程。因此，本文采用 SE(3) 的李群表示来建模空间变换。根据 [23]，SE(3) 的李代数与向量空间 \(R^6\) 同构，可通过指数映射与对数映射在 SE(3) 李群与 \(R^6\) 向量空间之间转换：

\[
T = \exp(\psi), \quad \psi = \log(T)
\]

其中 \(\psi \in R^6\)，\(T \in SE(3)\) 分别表示旋转向量和齐次变换矩阵。因此，在扩散过程中，本文将 SE(3) 变换表示为 6 维旋转向量，并在推理后映射回 SE(3)。

#### 物体节点

给定物体点云，首先使用最远点采样将其划分为 P 个 patch。为捕获待抓物体的空间与几何特征，本文采用 [25] 中用于 3D 断裂装配的预训练 VQ-VAE [24] 编码器，从单位球内归一化的 patch 中提取几何 token \(\{f_i^O\}_{i=1}^{P}\)。随后，将 patch 中心位置 \(\{c_i^O\}_{i=1}^{P}\)、全局物体尺度 \(s^O\) 及对应 patch 几何 token 拼接，构成物体节点：

\[
N^O = \{concat(c_i^O, s^O, f_i^O) \mid i=1,\ldots,P\} \in R^{P \times d^O}
\]

#### Link 节点

Link 节点同时编码每个 link 的几何属性，以及每个抓取中该 link 相对于物体全局坐标系的空间变换。为表示几何，本文对 link 点云使用 Basis Point Set，BPS，算法 [26]。具体而言，在单位球中均匀采样 B 个基点，并将每个 link 点云归一化到同一单位球。某个 link 的 BPS 特征是长度为 B 的向量，其中每个元素表示该点云与某个基点之间的最小距离。这种编码产生固定长度特征，不受输入点数影响，并可在 PyTorch 中高效实现，因此可扩展到多样灵巧手本体。

随后，基于 MLP 的编码器将 BPS 特征与 link 中心 \(\{c_i^R\}\) 和尺度 \(\{s_i^R\}\) 一起嵌入，得到每个 link 的几何嵌入。再将几何嵌入与各状态中 link 全局位姿相对于物体坐标系的 6D 旋转向量 \(\{\psi_i^R\}\) 拼接，形成图中的 link 节点：

\[
b_i = MLP(BPS(P_i^R), c_i^R, s_i^R), \quad i=1,\ldots,L
\]

\[
N^R = \{concat(\psi_i^R,b_i)\mid i=1,\ldots,L\} \in R^{L \times d^R}
\]

#### 边

由于刚体抓取中物体节点之间的相对位姿固定，因此不定义物体节点之间的边。相反，本文只在物体节点与 link 节点之间，以及 link 节点之间定义边。为获得更强的空间变换约束，图中采用完全连接。Link-link 边定义为两个 link 节点之间的相对空间变换。为保证不同物体之间的一致性，所有物体变换矩阵 \(\{T_i^O\}\) 采用单位旋转 \(R_i^O=I\)，并仅使用 patch 中心 \(\{c_i^O\}\) 作为平移。

边构建如下：

\[
T_i^{O,jR}=(T_i^O)^{-1}T_j^R,\quad T_j^{R,kR}=(T_j^R)^{-1}T_k^R
\]

\[
E^{OR}=\{\log(T_i^{O,jR})\},\quad E^{RR}=\{\log(T_j^{R,kR})\}
\]

其中 \(E^{OR}\) 和 \(E^{RR}\) 分别表示物体-link 边与 link-link 边。通过将物体 patch 与 link 之间的空间变换同丰富几何嵌入结合，T(R,O) Graph 被定义为：

\[
G=(N^O,N^R,E^{OR},E^{RR})
\]

它构成后续图扩散过程的基础结构。

### B. T(R,O) 图扩散模型

本文采用运行在 T(R,O) Graph 上的图扩散模型，使其能够从随机噪声生成多样且精确的抓取，并跨不同物体和本体泛化。DDPM [27] 通过随机过程逐步去噪高斯噪声来生成数据，但通常需要许多推理步骤。为提高推理效率，本文采用 DDIM [28]：它是 DDPM 的变体，可加速采样、控制生成多样性，并天然支持有条件生成。

#### 前向过程

如前所述，物体节点 \(N^O\) 在训练与推理过程中保持不变，而边 \(E^{OR},E^{RR}\) 必须满足节点之间的空间约束。因此，噪声注入和去噪仅作用于 link 位姿 \(\Psi^R=\{\psi_i^R\}\)。

给定由有效抓取导出的无噪图 \(G_0=(N_0^O,N_0^R,E_0^{OR},E_0^{RR})\)，首先按照方差调度 \(\beta_1,\ldots,\beta_T\) 对 link 位姿逐渐添加高斯噪声：

\[
p(\Psi_t^R|\Psi_{t-1}^R)=N(\sqrt{1-\beta_t}\Psi_{t-1}^R,\beta_t I)
\]

与 [27] 类似，可从无噪图直接采样任意时间戳 t 的 link 位姿。带噪 link 节点由带噪位姿向量与原始 BPS 嵌入拼接得到；随后根据节点之间的相对空间变换重新计算对应边，从而得到时间步 t 的带噪 T(R,O) Graph \(G_t\)。

#### 去噪模型

为从带噪 T(R,O) Graph \(G_t\) 预测 link 节点噪声，本文实现了 Transformer 式图去噪模型 \(\epsilon_\theta(G_t,t)\)。每个图元素首先编码为 token 表示，包括物体节点、link 节点、物体-link 边和 link-link 边，并加入时间戳嵌入 \(\phi(t)\)。随后，这些 token 经由一组图去噪层处理，其中 Object-Robot，OR，注意力和 Robot-Robot，RR，注意力机制迭代更新 link 节点和所有边。

在 OR 和 RR 注意力中，目标节点与连接边通过聚合相应源节点的信息进行优化。注意力模块将目标节点编码为 query，将源节点编码为 key/value，并用边特征调制注意力权重，从而在几何和相对位姿约束下更新图表示。最终，模型输出对 link 位姿噪声的估计。

#### 损失函数

扩散模型训练目标是通过变分下界最大化数据对数似然，可简化为最小化噪声预测误差。由于 link 位姿包含平移和旋转，本文分别对位置噪声和旋转噪声加权，训练目标可理解为：使模型从带噪 T(R,O) Graph 中准确恢复注入到 link 位姿上的噪声。

### C. T(R,O) Grasp 的训练与推理

#### 无条件抓取合成

无条件抓取合成不依赖额外初始手部状态。推理时，从随机高斯噪声初始化所有 link 位姿，并通过 DDIM 迭代去噪生成 T(R,O) Graph。随后，生成的 link 位姿被输入逆运动学求解器，恢复满足关节约束的可执行手部配置。由于 T(R,O) Graph 不依赖具体手部关节空间，该过程可适用于不同灵巧手本体。

#### 有条件抓取合成

有条件抓取合成允许利用初始手部状态、手掌方向或接触区域等外部引导。本文采用扩散引导策略，使生成过程在保持扩散多样性的同时满足条件约束。给定一个初始或期望手部状态，模型在去噪过程中引导 link 位姿靠近该状态，从而降低对不可行初始化的敏感性，并支持闭环场景中的连续抓取更新。

#### 逆运动学与可执行抓取

扩散模型预测的是各 link 的空间变换，但真实机器人手必须满足运动学链和关节限制。因此，本文使用基于 Pyroki 的逆运动学优化，将预测 link 位姿转换为可执行关节值。在此过程中，关节约束被显式施加，以提升生成抓取的物理可行性。

## IV. 实验

本文在仿真与真实机器人上评估 T(R,O) Grasp，并与现有方法进行比较。实验关注以下问题：T(R,O) Graph 是否能高效表示跨本体抓取；图扩散模型是否能产生高成功率和高多样性的抓取；该方法是否足够快以支持闭环灵巧操作。

### A. 实验设置

训练与评估覆盖多种灵巧手和大量物体。物体以点云形式输入，手部由 URDF 与 link 点云描述。本文评估无条件和有条件抓取合成，指标包括抓取成功率、推理时间、吞吐量和显存消耗。所有主要推理效率指标均在 NVIDIA A100 40GB GPU 上报告。

### B. 无条件抓取合成

在无条件设置下，模型仅根据物体点云和手部描述生成抓取，不依赖初始手部状态。T(R,O) Grasp 的平均成功率达到 94.83%，显著优于 GenDexGrasp、DFC 和 D(R,O) Grasp 等基线。相比 D(R,O) Grasp，T(R,O) Grasp 不需要以点到点距离矩阵构建高维交互特征，而是通过物体 patch 与 hand link 之间的变换图建模，因此显著降低内存消耗并提高推理速度。

### C. 有条件抓取合成

在有条件设置中，模型利用初始手部状态或外部引导合成抓取。T(R,O) Grasp 取得 92.33% 的成功率，并保持高吞吐量。与强依赖初始状态质量的 D(R,O) Grasp 不同，T(R,O) 的扩散生成过程可从带噪状态逐步修正为可行抓取，因此对不理想初始化更鲁棒。

### D. 效率分析

T(R,O) Grasp 的关键优势之一是效率。与 D(R,O) 的点到点表示相比，T(R,O) Graph 将关系建模在物体 patch 与手部 link 层级，减少了特征图规模和显存占用。实验表明，T(R,O) Grasp 的平均推理速度为 0.21 秒，吞吐量为每秒 41 个抓取；训练时间也显著短于 D(R,O) Grasp。论文图 1 中报告，相比 D(R,O) Grasp，本文方法在训练效率和推理吞吐量上都有明显提升。

### E. 闭环抓取

得益于快速推理速度，以及能够基于先前抓取位姿进行有条件合成，T(R,O) Grasp 可进行平均 5 FPS 的有条件抓取合成，使闭环抓取成为可能。给定离散时间序列 \(\{0,T,\ldots,nT\}\)，T(R,O) 使用上一时间戳的位姿 \(\Psi_{(i-1)T}^{R}\) 作为引导，预测当前时间戳的所有 link 位姿 \(\Psi_{iT}^{R}\)。

作者在 10 个未见物体上使用 Allegro 手进行闭环抓取评估。时间间隔设为 0.25 秒，总共 30 个评估步骤。每个间隔中，物体要么以恒定速度平移，速度为 5 cm/s；要么受到随机扰动，包括最高 1.25 cm 的平移噪声和每步最高 30 度的旋转噪声。结果显示，闭环 T(R,O) Grasp 在恒速运动下平均成功率为 89.3%，在随机扰动下平均成功率为 91.0%；而开环方法完全失败，成功率为 0%。这些结果证明 T(R,O) Grasp 可作为跨动态场景的通用闭环灵巧抓取框架。

### F. 真实机器人实验

作者在配备 Robotera XHand 和 LEAP Hand 的 uFactory xArm7 机器人上开展真实世界实验。物体点云观测由上方 Intel RealSense D435 相机获取。模型在 7,800 个抓取演示上训练，并在每种灵巧手的 10 个未见物体上评估 100 次抓取。推理阶段使用 FoundationPose [35] 跟踪物体位姿。

T(R,O) Grasp 在 XHand 和 LEAP Hand 上分别达到 91% 和 90% 的平均成功率，同时支持动态场景中的闭环抓取。论文图 7-9 展示了真实机器人设置、静态物体抓取和传送带动态抓取示例，说明该方法在新物体上具有较强鲁棒性和泛化能力。

## V. 结论

本文研究了高效且可泛化的灵巧抓取合成问题。作者指出了机器人中心、物体中心和交互中心表示的局限，并提出 T(R,O) Graph：一种统一、内存高效的物体-机器人交互表示，可跨本体泛化。基于该表示，作者提出 Transformer 图扩散模型，同时支持有条件和无条件抓取生成，并支持闭环操作。

仿真和真实硬件上的大量实验表明，T(R,O) Grasp 在成功率上优于所有基线，同时具备高效率和闭环操作能力。这项工作不仅推进了抓取方法，也为大规模灵巧抓取基础模型奠定了基础。未来方向包括更广泛的操作任务、多模态反馈和大规模预训练。

## 附录 B. 网络架构

### 1. VQ-VAE 编码器

在 T(R,O) Graph 中，作者利用 [25] 的预训练 VQ-VAE 编码器划分物体 patch，并提取对应几何 token。物体点云首先归一化到单位球内，再由预训练编码器编码为 \(P=25\) 个局部几何特征 \(\{f_i^O\}_{i=1}^{P}\)，以及对应的 patch 中心坐标 \(\{c_i^O\}_{i=1}^{P}\)。

### 2. BPS 编码器

由于每个 link 的点云数量不同，作者使用 Basis Point Set，BPS，算法 [26] 将每个 link 点云编码为固定长度几何特征。具有 L 个 link 的灵巧手点云定义为各自局部坐标系中的 \(\{P_i^R\}_{i=1}^{L}\)。首先将所有点归一化到单位球；随后在单位球中随机采样 \(B=124\) 个点作为所有 link 点云的基点集合。BPS 特征由归一化 link 点云与基点集合之间的最小距离构成，用于表示 link 几何，并进一步编码为 link 节点。

### 3. 图去噪层

为从带噪 T(R,O) Graph 预测 link 节点噪声，作者采用由 \(N=6\) 层组成的图去噪器，每层包含一个 OR-attention 模块和一个 RR-attention 模块。注意力机制聚合图节点和边的信息来更新其表示。

## 附录 C. 跨本体零样本泛化

由于 T(R,O) Grasp 只在有限种灵巧手上训练，直接在完全未见手上进行零样本实验并不可行。因此，作者从现有手部创建派生本体，以可控方式近似跨本体零样本泛化。具体做法是修改 Allegro、Barrett 和 ShadowHand 的 link 长度与关节限制。为评估本体相似度，作者定义 link 对齐度 \(S_L\) 和关节重叠度 \(S_J\)。

作者在 Allegro、Barrett 和 ShadowHand 的原始本体上训练 T(R,O) Grasp，并在其派生本体上评估。结果显示，当测试手与训练手相似度不低于 0.5 时，T(R,O) Grasp 可取得超过 70% 的成功率，说明其在具有相近几何结构的灵巧手本体之间具备较强零样本能力。当前零样本性能仍受训练本体数量有限的约束；这表明，如果在大规模本体数据集上训练，T(R,O) 有潜力扩展为具有强零样本泛化能力的灵巧抓取基础模型。

## 附录 D. 更多实现细节

### 1. T(R,O) Graph 构建

对于物体节点，预训练 VQ-VAE [24] 将物体点云编码为 25 个 64 维 token，因此物体节点维度为 \(N^O \in R^{25 \times (3+1+64)}\)。对于 link 节点，BPS 特征与 link 中心、尺度一起被嵌入为 128 维。为支持不同 link 数量本体之间的并行计算，link 节点被零填充为 \(N^R \in R^{25 \times (6+128)}\)。因此，物体-link 边和 link-link 边分别为 \(E^{OR} \in R^{6 \times 25 \times 25}\) 与 \(E^{RR} \in R^{6 \times 25 \times 24/2}\)。

### 2. 训练

训练时，噪声方差采用线性调度，从 \(\beta_{min}=1 \times 10^{-4}\) 到 \(\beta_{max}=0.02\)，总扩散步数 \(T=1000\)。T(R,O) Grasp 使用 Adam 优化器训练 300 个 epoch。初始学习率设为 \(1 \times 10^{-4}\)，每 20 个 epoch 乘以 0.8 衰减。位置与旋转噪声损失权重均设为 1.0。

### 3. 推理

对于无条件和有条件抓取合成，作者均采用 DDIM [28] 扩散策略，采样 \(M=20\) 步进行推理。为鼓励抓取多样性，推理过程中设置 \(\lambda=0.2\) 注入随机噪声。在有条件抓取合成中，梯度引导强度按正弦调度设置，使扩散早期保留生成多样性，后期鼓励生成抓取遵循方向引导。

## 附录 F. 讨论与局限

T(R,O) Grasp 是一种用于跨本体灵巧抓取合成的高效图扩散模型。它在抓取成功率和推理速度上显著优于所有基线，并具有更低内存消耗。借助扩散模型的性质，T(R,O) Grasp 支持无条件和有条件抓取合成，例如基于初始位姿或接触区域进行条件化。

然而，T(R,O) Grasp 仍存在一定局限。由于训练中的灵巧手集合有限，模型难以在完全未见本体上直接进行零样本抓取合成。因此，未来工作应将 T(R,O) Grasp 扩展到更广泛的灵巧手集合，涵盖更多样、更全面的几何结构。此外，T(R,O) Grasp 网络没有在扩散过程中显式考虑物体-机器人碰撞或关节限制约束，可能生成物理上不可行的抓取。因此，将碰撞检测和运动学约束感知整合进扩散过程，可进一步提升抓取合成准确性。最终，作者设想 T(R,O) 可成为机器人-物体交互的基础表示，支持策略学习和规划等多种下游应用。
